# Contributing to Multi-Tier-App-Troubleshooting-Playground

We welcome contributions that improve troubleshooting workflows, documentation clarity, and reproducibility.

## How to Contribute

- **Fork the repository** and create your branch from `main`.
- **Write clear commits** that explain the change (e.g., `fix: update screenshot sync script`).
- **Add documentation** for any new scripts, workflows, or troubleshooting steps.
- **Test your changes** locally before submitting a pull request.
- **Open a Pull Request (PR)** with a detailed description of:
  - What problem you solved
  - How you solved it
  - Any screenshots or logs that help reviewers

## Code Style

- Use **2 spaces** for indentation.
- Keep scripts modular and reproducible.
- Document commands and scripts with inline comments.

## Reporting Issues

- Use the **Issues tab** to report bugs or request features.
- Include steps to reproduce and any relevant logs.

## Community Guidelines

- Be respectful and constructive.
- Focus on clarity and reproducibility.
- Share troubleshooting insights that empower others.

Thank you for helping make this project a reliable troubleshooting playground!
